# 데이터 라벨링 단계별 작업 계획 (Phase 1: 실 라벨 ≥ 1만)

> **목적**: Donut 값 인식을 "사용 불가(0%)"에서 "논문 수준(F1 0.9대)"으로 끌어올리는 **결정적 레버 ①**.
> 논문은 ~13,000 numeric 크롭으로 F1 0.963을 냈다. 현재는 **1,975장(논문의 1/7)** — 이걸 **≥ 10,000**으로.
>
> CVAT 조작법(값 속성·OBB·export)은 [`CVAT_LABELING_GUIDE.md`](CVAT_LABELING_GUIDE.md) 참고.
> 이 문서는 그 위에서 **"1만 장까지 어떻게 굴릴지"**의 운영 계획이다.

---

## 0. 목표 정량화 (현재 → 목표)

현재: **50 도면 → 1,975 크롭** (도면당 ~40 크롭).
목표 10,000 크롭 → **추가 ~8,025 크롭 ≈ 추가 도면 ~200장 (총 ~250장)**.

| 클래스 | 현재 | 목표 | 추가 필요 | 비고 |
|---|---:|---:|---:|---|
| Dimension | 1,284 | 5,000 | +3,716 | 자연적으로 다수 — 자동 확보됨 |
| Surface_Roughness | 292 | 1,500 | +1,208 | 소수 — **의도적 확보** |
| GD&T_FCF | 180 | 1,500 | +1,320 | 소수 — **의도적 확보** |
| Datum | 149 | 1,000 | +851 | 소수 — **의도적 확보** |
| Hole_Callout | 70 | 800 | +730 | 희소 — **의도적 확보** |
| Section | 0 | 200 | +200 | 현재 라벨 0건 — 신규 |
| **합계** | **1,975** | **10,000** | **+8,025** | |

> **핵심 원칙**: 단순 총량이 아니라 **소수 클래스 절대량 확보**. 지금 65% Dimension 편중이라,
> 도면을 그냥 더 모으면 Dimension만 늘고 GD&T/Datum/Hole은 여전히 모자란다.
> → **소수 클래스가 많은 도면(정밀 가공품·조립도)을 우선 소싱**한다.

---

## 1. 단계 개요 (파이프라인)

```
[A] 도면 소싱        원본 PDF ~200장 추가 확보 (소수 클래스 풍부한 것 우선)
     │
[B] 전처리           rasterize_pdf.ipynb → 300 DPI PNG → view YOLO로 뷰 크롭 자동 분할
     │
[C] 모델 보조 사전라벨  학습된 element YOLO-OBB로 박스 자동 생성 → CVAT에 사전 주입  ★효율 핵심
     │
[D] 사람 라벨링/교정   박스 보정 + class + value(+ 구조화 필드) 입력
     │
[E] QA·검수          format 검증 + 소수 클래스 더블체크 + pending→reviewed
     │
[F] 변환·학습        cvat_to_donut.py → split → 재학습 → field-F1 평가
     │
[G] 능동학습 루프      오답·불확실·희소 케이스를 다음 배치로 우선 (A로 복귀)
```

---

## 2. [A] 도면 소싱 (~200장 추가)

- **출처**: 사내 도면 아카이브 / 부품 공급사 도면 / 공개 기계도면 세트.
- **선별 기준** (소수 클래스 보강이 목적):
  - GD&T·Datum이 많은 **정밀 가공/검사 도면** 우선.
  - Hole_Callout이 많은 **플레이트·브래킷·하우징** 도면.
  - Section 표시가 있는 **단면도 포함 도면**.
- **다양성**: 회사·템플릿·표제란 양식·폰트가 **겹치지 않게** (일반화 성능 ↑).
- 산출: `data/raw_pdf/` 에 추가 (현재 50장 → ~250장).

---

## 3. [B] 전처리 (자동)

1. **래스터화**: `detection/rasterize_pdf.ipynb` (300 DPI) → `data/drawings_pages/*.png`.
2. **뷰 분할(자동)**: 학습된 view YOLO(`detect`, mAP50 0.994)로 뷰 크롭 생성 → `data/view_crops/`.
   - 뷰 검출은 사실상 완성형이라 **사람 개입 거의 불필요**(스팟 체크만).
3. 뷰 크롭이 **element 라벨링의 입력 단위**가 된다 (작은 글자 해상도 확보).

---

## 4. [C] 모델 보조 사전 라벨링 (★ 처리량의 핵심)

**from-scratch로 박스를 다 그리면 느리다.** 이미 학습된 element YOLO-OBB(mAP50 0.918)로
박스를 **미리 그려** CVAT에 넣고, 사람은 **틀린 것만 고친다**.

- 방법 1 (간단): 각 뷰 크롭에 element YOLO 추론 → CVAT "CVAT for images 1.1" XML로 변환 →
  CVAT job에 **annotations upload**(import). (박스만; value는 비움)
- 방법 2 (고급): CVAT **Nuclio serverless** 모델 등록 → job에서 "Automatic annotation" 실행.
- 효과: 박스 그리기(30~60초/개) → **박스 확인·미세보정(5~15초/개)**. 라벨링 시간 **2~4배 단축**.

> 사전라벨 박스는 **검출용**이다. 값(value)·구조화 필드는 사람이 입력해야 한다(아래 [D]).

---

## 5. [D] 사람 라벨링 / 교정 (작업 규약)

CVAT에서 박스마다: **① 박스 정렬 보정 → ② class 확인 → ③ value 입력 → ④ (선택) 구조화 필드**.
세부 조작은 `CVAT_LABELING_GUIDE.md`. 여기서는 **일관성 규약**만:

- **class 이름**: `element.yaml`과 **정확히 일치**(`Dimension`, `GD&T_FCF`, `Datum`, `Surface_Roughness`, `Section`, `Hole_Callout`).
- **value 표기 규약(통일 필수 — 안 하면 같은 값이 다르게 라벨됨)**:
  | 항목 | 규칙 | 예 |
  |---|---|---|
  | 지름 | `Ø` 기호 사용(영문 O 금지) | `Ø65` |
  | 공차 | `±` 또는 `+a -b` | `90±0.1`, `Ø20 +0.05 -0.02` |
  | 수량 | `N×`(곱셈기호 ×, x 혼용 금지) | `4× Ø8` |
  | 거칠기 | `Ra` + 공백 + 수치 | `Ra 1.6` |
  | 공백 | 기호와 수치 사이 **규칙 고정**(전부 붙이거나 전부 띄우거나) | 팀 내 1택 |
- **구조화 필드(Phase 2 통합)**: CVAT 라벨에 `quantity/nominalValue/upperLimit/lowerLimit` 등
  속성을 추가해 **분해값을 같이 입력**하면 재작업이 없다. (가이드: [`구조화 스키마 가이드`](Element_Donut_구조화스키마_수작업annotation_가이드.md))
- **모호/불량 박스**: 읽을 수 없으면 value 비우고 `status=skip` 표시(쓰레기 라벨 방지).

---

## 6. [E] 품질관리 (QA)

라벨 품질 = 모델 품질. 다음을 **배치마다** 수행:

1. **자동 형식검증 스크립트**(작성 권장): 각 `{class:value}`에 대해
   - 빈 value / `<unk>` 유발 문자 / class-value 불일치(예: Roughness인데 `Ø…`) 플래그.
   - 표기 규약 위반(Ø vs O, × vs x) 검출.
2. **소수 클래스 더블체크**: GD&T·Datum·Hole·Section은 **2인 검수**(오류 영향이 큼).
3. **상태 흐름**: `pending → reviewed`(현재 1,974건 pending도 이 흐름으로 정리).
4. **인터-어노테이터 스팟체크**: 무작위 5%를 교차 검수해 규약 합치도 확인.

---

## 7. [F] 변환 · 학습 · 평가

1. CVAT → **"CVAT for images 1.1"** XML export.
2. `python detection/cvat_to_donut.py --xml <export.xml> --images ../data/view_crops`
   → `data/elements/{images,labels}` 생성 (`rectify_obb`로 추론과 동일 정렬).
3. `donut_training_elements.ipynb` split → 재학습(bf16, task `<s_element>`).
4. **평가는 field-level P/R/F1**(부분점수) + 실검증셋으로만. exact만 보지 말 것.
5. (보강) 합성 `synth_elements.py`로 부족 클래스 보충 — **실데이터 우선, 합성은 보조**.

---

## 8. [G] 능동학습 루프 (배치 전략)

**한 번에 1만 장을 다 하지 말고**, 배치로 나눠 "라벨→학습→약점 파악→다음 배치 집중"을 반복한다.

| 배치 | 누적 크롭 | 초점 | 산출 |
|---|---:|---|---|
| B1 | ~3,000 | 기존 1,975 검수 + 소수 클래스 1차 보강 | 1차 재학습, baseline F1 |
| B2 | ~6,000 | B1 모델이 **틀리거나 불확실한** 케이스 우선 라벨 | F1 추적 |
| B3 | ~10,000 | 잔여 약점·희소 패턴(Section, 복합 GD&T) 집중 | 목표 도달 |

- **능동학습 선별**: B1 모델을 미라벨 뷰 크롭에 돌려 **낮은 conf / 오분류** 박스를 다음 배치로.
- **조기 신호**: 배치마다 field-F1을 찍어 **곡선이 꺾이는지** 본다(증가 정체 시 데이터 다양성 점검).

## 9. 처리량 · 일정 추정

- 모델 보조 시 **~10~15초/박스**(보정+값입력). 추가 8,025박스 → **약 25~35 작업시간**.
- 인력별 일정(QA 포함, 실효 가동 고려):
  | 인력 | 라벨링 | + QA·검수 | 총 |
  |---|---|---|---|
  | 1명 | ~5일 | +2일 | **~7 영업일** |
  | 2~3명 | ~2일 | +1일 | **~3 영업일** |
- 병목은 **소수 클래스 도면 소싱**일 수 있음 → [A]를 먼저/병행 착수.

## 10. 마일스톤 & 완료 기준

- [ ] **M0** 스키마·표기 규약 동결 + CVAT 라벨 속성 구성(class+value+구조화 필드)
- [ ] **M1** 도면 ~250장 확보·래스터화·뷰 크롭 생성
- [ ] **M2** element YOLO 사전라벨 주입(모델 보조 세팅 완료)
- [ ] **M3** 배치 B1(~3,000) 라벨+검수 → 1차 재학습 → baseline field-F1
- [ ] **M4** 배치 B2(~6,000) — 능동학습 기반 약점 보강
- [ ] **M5** 배치 B3(~10,000) — 목표 도달, 소수 클래스 절대량 충족
- [ ] **M6** 최종 재학습 → field-F1 평가(논문 0.9대 근접 확인)

**완료 기준**: 총 ≥10,000 크롭 + **클래스별 최소 절대량**(GD&T·Roughness ≥1,500 / Datum ≥1,000 / Hole ≥800 / Section ≥200) 충족 + 형식검증 통과율 ≥99%.

## 11. 리스크 & 대응

| 리스크 | 영향 | 대응 |
|---|---|---|
| 소수 클래스 도면 부족 | GD&T/Datum/Hole 목표 미달 | 소싱 단계에서 해당 도면 우선 확보 + 합성 보강 |
| 표기 규약 불일치 | 같은 값이 다른 라벨 → 학습 혼란 | M0에서 규약 동결 + 형식검증 자동화 |
| 사전라벨 과신 | YOLO 오검출을 그대로 승인 | 박스 **확인 의무**, 소수 클래스 2인 검수 |
| 통짜 value 누적 | Phase 2 재작업 발생 | 처음부터 **구조화 필드 동시 입력**(②와 통합) |
| exact만 평가 | 진전이 0으로 묻힘 | **field-level F1**로 부분점수 추적 |

---

> **요약**: ①데이터 확충은 **"소수 클래스 절대량"** 싸움이다. ❶소수 클래스 풍부한 도면 소싱 →
> ❷뷰 자동분할 → ❸element YOLO **사전라벨**로 속도 확보 → ❹사람은 보정+값(+구조화 필드) →
> ❺배치+능동학습으로 약점 집중. **모델 보조 + 배치 반복**이 1만 장을 현실적 일정 안에 끝내는 열쇠다.
