## B. 수동 작업 큐 (37건) — 라벨만으로 못 고침

### B-1. 회전(180°) 크롭 → 회전 후 재라벨 (~16건)
**증상**: 선두 `×`/`X` + 말미 `Ⓛ`, 뒤집힌 숫자(`ε0'0`,`LO'O`). 이미지가 180° 뒤집혀 있어 OCR 가 거꾸로 읽음.
**검증**: 1783(`X10.0X`)=실제 `◎0.01X`, 2084(`× 0.03 Ⓛ`)=실제 `◎0.03X` (둘 다 회전 확인).
**조치**: 크롭 180° 회전(rectify) 후, 올바른 `<기호><공차><datum>` 으로 재라벨.

`1783, 1790, 2084, 2087, 2110, 2154, 2184, 2248, 2249, 2250, 2251, 2271, 3038, 3044, 3046` (+ `× 0.1 Ⓛ` 류)

### B-2. 선두 기하기호 누락 → 라벨에 기호 추가 (~9건)
**증상**: 이미지엔 기하기호가 있으나 라벨이 공차부터 시작(`0.010A`). **이미지는 정상**.
**검증**: 1386(`0.010A`)=이미지에 ◎류 기호 존재 → `◎0.010A` 로 보정 필요.
**조치**: 각 크롭 이미지를 보고 선두 기호(◎/⊥/∥/⊕ 등)를 라벨 앞에 추가.

`1386, 1776, 1839, 1857(Z?), 1904, 1906, 1920, 2530, 2743`

### B-3. 다중요소·병합 크롭 → 재크롭 (~7건)
**증상**: 한 패치에 노트/치수/나사가 섞임(`\n` 또는 여러 토큰). 라벨 정리로 안 되고 **재검출/재크롭 필요**.

`2763(GRIND 32 KMS…), 2767(L01,10…), 2768(L100,0…), 2819(212.0…), 2959(L12.10 … 병합), 1978(L0∥), 3202(나사+GD&T 병합)`

### B-4. 모호 기호 판별 (±/= 오독, ~4건)
**증상**: `±`/`=` 가 기하기호 오독일 가능성. 검증: 2121(`±0.05`)=이미지의 두-줄 기호를 `±` 로 오독.
**조치**: 이미지 보고 실제 기호로 교정(또는 FCF 아니면 재분류).

`2121, 2333, 1483(Ø…±… 복합), 1640(=0,05-)`

## C. 권장 순서 & 기대효과

1. **B-3 재크롭**(노이즈 가장 큼) → 2. **B-1 회전 보정** → 3. **B-2 기호 추가** → 4. **B-4 판별**.
2. 정제 후 **재학습**해야 효과 발현(라벨 수정은 현재 체크포인트 점수를 바꾸지 않음).
3. 근거: 논문 B 는 1,406 크롭으로 F1 0.963 — **양이 아니라 이런 품질**이 격차의 핵심([`성능미달`](Element_Donut_성능미달_원인_및_해결방안.md)).
